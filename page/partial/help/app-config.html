App Config JSON

NoSQL:IDE is fully driven by JSON doc files. The main JSON doc is app.config.json. This JSON doc stores all the necessary information for configuring, loading, running and connecting to your databases. To edit your app.config.json, click the config settings Gears icon in top/left of app UI.

Following is an example of a valid app.config.json that can connect to 2 different Firestore databases. You can connect to any number of different databases.

Details for each attrib are below the screenshot.

fontSize
This sets the default size of the text in the editor.

colorTheme
This sets the default theme colors displayed in both the app UI and in the editor.

startupDb
This sets the default database to load in the DbChooser dropdown and collection links to display in the UI main left nav bar.

schemaNamespaceUrl
This sets the default namespace for your document schemas. All json-schema.org schemas need a unique namespace.

db
This object stores the database specific info for each of your different databases.

db.yourdatabasename
Enter one of these objects within the db object for each of your different database. Ofc replace yourdatabasename text with the actual name of your database.

db.yourdatabasename.toolsPlugin
This sets the correct DbTools plugin to load for your specific database. As of 7/20/19: Only Google Firestore database is supported. More DbTools plugins are in development.

db.yourdatabasename.dbUserEmail
This sets the user account credentials name/email used to connect to your database. This account must have an appropriate role in your database with enough permissions to read/write to your collections. It does not have to be a full admin account.

db.yourdatabasename.dbUserPass
This sets the user account credentials password. NOTE: Nothing in this app.config.json is transmitted out or copied/stored anywhere outside your local computer. Your account creditials are local and safe.

db.yourdatabasename.softDeleteAttrib
This sets the name of the collection attribute used to mark your documents as soft deleted. Set this to null if you do not want to use soft deleted documents. NoSQL:IDE queries automatically honor this soft delete attrib. You do need to have/set this attrib to false when creating new docs, however there is no need to enter it in any queries. Soft deleting is better than hard deletes for soooo many reasons. NOTE: A new cleanup feature to purge (hard delete) ranges of soft deleted documents is in development.

db.yourdatabasename.config
This is your database configuration details. You must get this specific JSON from your database provider. The screen cap above shows a Google Firestore config. Please consult the documentation for each different database.

db.yourdatabasename.collections
These are the names of each collections in your database you wish to access. Leave out any collection you don't want/need to edit.
